# Phase 2 Demo 演示路径

## 标准演示路径（端到端）

### 前置条件

1. 使用 Node 20.x 运行 `pnpm dev`
2. 浏览器打开 `http://localhost:3000`
3. 首次使用需注册账号（输入用户名即可）

### 路径步骤

| 步骤 | 操作 | 页面/视图 | 预期结果 | 失败兜底 |
|------|------|-----------|----------|----------|
| 1 | 注册/登录 | `/workspace` (AuthGate) | 进入工作区，左侧显示 Dock/Entries/Review 导航 | 若已有账号，直接选择已有账号登录 |
| 2 | 快速输入 | Dock 视图底部输入框 | 输入文字后按 Enter，条目出现在 Dock 列表，自动选中 | 若未出现，点击"记录"按钮；检查控制台错误 |
| 3 | 展开输入 | 点击输入框旁展开按钮 | 弹出展开编辑器模态框，可输入长文本 | 按 Esc 关闭重试 |
| 4 | 生成建议 | 选中 pending 条目，点击"生成建议" | 右侧详情区显示分类、标签建议（含 reason）、动作建议 | 若条目不是 pending 状态，先"恢复到 Dock" |
| 5 | 编辑标签 | 在整理区点击标签建议或手动添加 | 标签出现在条目的 userTags 中；可忽略建议（✕） | 手动输入标签名后按 Enter |
| 6 | 接受归档 | 点击"接受归档" | 条目从 Dock 消失，Entries 列表新增一条 | 切换到 Entries 视图确认 |
| 7 | 忽略条目 | 选中 suggested 条目，点击"忽略" | 条目状态变为 ignored | 在 Dock 中可"恢复到 Dock" |
| 8 | 浏览归档 | 切换到 Entries 视图 | 显示已归档条目列表，顶部有筛选栏 | 若列表为空，回到 Dock 继续归档 |
| 9 | 筛选 | 使用 status/type/tag/project 下拉筛选 | 列表按条件过滤，计数显示 filtered/total | 点击"清除筛选"重置 |
| 10 | 查看详情 | 点击归档条目 | 右侧显示详情：标题、内容、标签、项目、动作 | 确认条目已被选中（蓝色高亮） |
| 11 | 编辑归档 | 点击"编辑"按钮 | 进入编辑模式，可修改内容、标签、项目 | 点击"取消"退出编辑 |
| 12 | 重新整理 | 点击"重新整理"按钮 | 条目回到 Dock，状态变为 reopened | 切换到 Dock 确认条目出现 |
| 13 | 重新建议 | 选中 reopened 条目，点击"重新生成建议" | 生成新建议，可再次归档 | 确认旧建议已清空 |
| 14 | Chat 模式 | 点击 Sidebar 顶部 Chat 按钮 | 主区域切换为 Chat 输入界面 | Classic 按钮取消高亮 |
| 15 | Chat 输入 | 输入一句话，点击"继续" | 进入确认页，可补充上下文 | 返回修改 |
| 16 | Chat 标签 | 点击"添加标签"，输入标签 | 标签显示在列表中，可删除 | 可跳过标签直接确认 |
| 17 | Chat 入 Dock | 点击"确认入 Dock" | 显示成功反馈，可跳转 Dock 查看 | 点击"继续记录"再试 |
| 18 | Chat 结果 | 点击"去 Dock 查看" | 切换到 Classic 模式 Dock 视图，列表顶部可见 chat 来源记录 | 手动切换到 Dock 视图 |
| 19 | Review 概览 | 切换到 Review 视图 | 显示统计卡片（已归档/待整理/标签数）+ 最近归档列表 | 若无数据，先填充 demo 数据 |

### 快速 Demo 数据填充

访问 `/seed` 页面，点击"填充 Demo 数据"按钮，自动创建：
- 8 条 Dock 条目：3 条 archived + 1 条 reopened + 1 条 ignored + 1 条 suggested + 2 条 pending
- 其中 2 条为 chat 来源（灵感条目 + 周末计划），其余为 text 来源
- 4 条归档条目（meeting/reading/idea/task），每条与对应 DockItem 通过 sourceDockItemId 关联
- 7 个标签（产品/技术/学习/项目管理/生活/性能/工作）

**关键特性**：
- 归档条目点击"重新整理"可回到 Dock（reopened），因为 sourceDockItemId 指向真实的 DockItem
- Entries 中归档条目对应不同状态的 DockItem，status 筛选可展示功能
- chat 来源条目与 text 来源条目共享同一 Dock 列表和状态流转
- reopened 条目可点击"重新生成建议"继续整理
- 清除操作仅清理当前用户数据，不影响其他用户

### 演示话术要点

1. **"Atlax 帮你接住一切"** — 快速记录灵感，系统自动整理
2. **"Classic + Chat 双入口"** — 两种方式记录，同一主线整理
3. **"建议可解释，用户优先"** — 每条建议有 reason，用户标签覆盖系统建议
4. **"归档不是终点"** — 随时可以重新整理已归档内容
5. **"按你想要的方式查看"** — 四维筛选，快速找到需要的内容

### 容易翻车的流程

| 风险点 | 原因 | 兜底方案 |
|--------|------|----------|
| 空库演示 | 无数据导致空状态 | 先访问 `/seed` 填充数据 |
| 条目状态不对 | 演示时误操作改变状态 | 用"恢复到 Dock"或"重新整理"重置 |
| 筛选后看不到数据 | 筛选条件过严 | 点击"清除筛选" |
| 刷新后丢失登录 | localStorage 被清除 | 重新注册同一用户名即可 |
| Chat 提交后看不到 | 未切换到 Dock 视图 | 点击"去 Dock 查看"自动切换 |
